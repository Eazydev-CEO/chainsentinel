# REST API

Base URL: `/api/v1/` · Interactive reference: **`/api/v1/docs/`** (Swagger UI, generated by
drf-spectacular) · Raw schema: `/api/v1/schema/`.

## Authentication

### Browser (dashboard)
Cookie-based JWT. `POST /auth/login/` sets HttpOnly `cs_access` (15 min) and `cs_refresh`
(14 days, path-restricted to `/api/v1/auth`) cookies. Unsafe methods require the
`X-CSRFToken` header echoing the `csrftoken` cookie (prime it via `GET /auth/csrf/`).
`POST /auth/refresh/` rotates the pair; the old refresh token is blacklisted.

### Integrations (API keys)
```
curl -H "X-Api-Key: cs_ab12cd34_…" https://your-host/api/v1/events/?page_size=10
```
Keys are workspace-bound. Scopes: `read` (safe methods) and `write` (mutations, acts as
admin). Keys are created by the workspace **owner** at `/api/v1/api-keys/` — the secret is
returned exactly once.

## Workspace context

Workspace-scoped endpoints need the active workspace via **one** of:
`X-Workspace-Id: <id>` header · `?workspace=<id>` query param · `"workspace": <id>` body
field. API keys imply their own workspace automatically.

## Error envelope

Every error responds with:
```json
{ "error": { "code": "validation_error", "message": "Validation failed. Check the details.", "details": { "address": ["Not a valid EVM address."] } } }
```

## Pagination, filtering, throttling

* Page-number pagination: `?page=`, `?page_size=` (max 100), responses carry
  `count/next/previous/results`.
* `django-filter` + search (`?search=`) + ordering (`?ordering=-created_at`) on list
  endpoints; see Swagger for per-endpoint parameters.
* Global throttles: authenticated 600/min, anonymous 60/min, plus scoped rates on sensitive
  endpoints. 429 responses include `wait_seconds`.

## Example: create a wallet monitor

```bash
curl -X POST "https://host/api/v1/wallet-monitors/?workspace=1" \
  -H "X-Api-Key: cs_…" -H "Content-Type: application/json" \
  -d '{
    "name": "Treasury",
    "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    "chain": "ethereum-sepolia",
    "direction": "both",
    "event_types": ["native_transfer", "erc20_transfer", "approval"],
    "large_tx_threshold_wei": "1000000000000000000",
    "severity": "high"
  }'
```

Response `201` returns the monitor with the checksummed address and effective
`required_confirmations`.
